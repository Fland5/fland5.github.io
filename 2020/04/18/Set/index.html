Set:无序，元素不重复。
对象相等性：引用堆上同一对象的两个引用是相等的。要使两个不同对象视为相等，必须覆盖Object继承下来的hashCode（返回该对象内存地址）方法和equals（对象比较）方法。
HashSet:基于Hash表，按Hash值存取数据。同样通过hashCode()与equals(Object o)方法判断元素是否重复。
元素的哈希值是通过元素的hashCode方法来获取的,HashSet首先判断两个元素的哈希值，如果哈希值一样，然后比较equals方法,如果equals结果为true，HashSet就视之为同一个元素，不再存储。
哈希值相,equals为false的元素在哈希表中是怎样存储呢？
同哈希值同列！在同样的哈希值向下顺延（可以认为哈希值相同的元素放在同一个哈希桶中），如下图中e1、e2、e3就属于这种情况。

TreeSet:有序，无重复，红黑树（一种特定类型的二叉树，左小右大）的数据结构，默认对元素进行自然排序（String升序排序:String实现了一个名为Comparable的接口，并重写了该接口的compareTo方法，因此String具有可比性）。
TreeSet指定排序规则的两种方式

  1. Comparable
    compareTo(Object o)—元素自身具备比较性:元素需要实现Comparable接口，覆盖compareTo方法。
  2. Comparator
    compare( Object o1, Object o2 )—给容器传入比较器:定义一个类实现Comparator接口，覆盖compare方法。

LinkedHashSet：保存插入顺序。
命名指向数据结构
array->数组。
linked->链表。
hash->hash表。
tree->树。
