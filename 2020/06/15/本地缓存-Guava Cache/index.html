本地缓存-Guava Cache
Guava 是一个 Google 开发的 基于 Java 的类库集合的扩展项目，包括集合 [collections] 、缓存 [caching] 、原生类型支持 [primitives support] 、并发库 [concurrency libraries] 、通用注解 [common annotations] 、字符串处理 [string processing] 、I/O 和验证的实用方法等等。
其中Guava Cache是一套采用全内存方式、线程安全的本地缓存项目。

  4. 缓存回收

    1. Size-based Eviction(基于容量回收)
      缓存容量设为定值，触及回收标准时，将通过LRU算法进行回收。
      CacheBuilder.maximumSize(long)设置。
    2. Timed Eviction(基于时间回收)
      1. CacheBuilder.newBuilder().expireAfterAccess(30,TimeUnit.MINUTES)
        缓存在30分钟内未被读写则会被回收。
      2. CacheBuilder.expireAfterWrite(30,TimeUnit.MINUTES)
        缓存在30分钟内未被更新则会被回收。
    3. Reference-based Eviction(基于引用类型回收)
      1. 3种引用概念
        强引用：最常见的引用方式，变量赋值就是强引用，只要引用关系存在，引用的对象就不会被垃圾回收。
        弱引用：在系统进行垃圾回收时，弱引用对象总会被回收。
        软应用：强度介于强引用与弱引用间，在内存空间不足，系统内存溢出前，软引用关联对象才会被回收。
      2. CacheBuilder.weakKeys()
        以弱引用的方式存储键值，当没有强引用或软引用指向某键值时，允许系统在进行垃圾回收时回收缓存项。
      3. CacheBuilder.weakValues()
        以软引用方式包装缓存值，只有在内存接近内存溢出时，系统会按照全局LRU算法进行垃圾回收。
    4. 手动回收
      单个回收，Cache.invalidate(key)
      批量回收，Cache.invalidateAll(keys)
      全部回收 Cache.invalidateAll()
    5. 方法调用
      cache.cleanUp() 缓存清理
      CacheBuilder.recordStats() 开启统计功能
      cache.stats() 运行统计信息：
      hitRate(), 缓存命中率。
hitCount()，缓存命中次数。
loadCount()，新值加载次数。
requestCount()，缓存访问次数，是命中次数和非命中次数之和。
averageLoadPenalty(),加载新值时的平均耗时，以纳秒为单位。
evictionCount(), 除了手动清除意外的缓存回收总次数。
    6. 并发设置
      类似于ConcurrentHashMap中的并发级别，GuavaCache通过该值将内部数据结构拆分成固定数量的段，缓存数据在段中存储，每个段有自己的写锁，段与段之间互不影响，所以并发级别越大，分的段越多，并发能力越强，如果服务QPS较大，可以适当调大该值。
      CacheBuilder.newBuilder().concurrencyLevel(30)
