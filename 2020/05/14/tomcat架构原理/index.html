tomcat架构原理

发表于 2020-05-14 | 分类于 Web Server
  1. 总体结构

    server -> (多个)service -> (多个)Connector + (单个)Container
  2. tomcat目录结构
  3. Lifecycle接口控制tomcat中组件的生命周期
    组件生命周期由包含它的父组件控制。
    各级组件层层实现Lifecycle接口，以被最高级组件为Server所控制。
  4. Connector组件
    Connector用于处理连接相关的事情，并提供Socket与Request和Response相关的转化。
    Connector最底层使用的是Socket来进行连接的，Request和Response是按照HTTP协议来封装的，所以Connector同时需要实现TCP/IP协议和HTTP协议。
  5. Container组件
    Container用于封装和管理Servlet，以及具体处理Request请求。

Container 容器的设计用的是典型的责任链的设计模式，它有四个子容器组件构成，分别是：Engine、Host、Context、Wrapper，这四个组件是父子关系，Engine 包含 Host,Host 包含 Context，Context 包含 Wrapper。
通常一个 Servlet class 对应一个 Wrapper。
