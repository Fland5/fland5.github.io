Mysql存储引擎-MyISAM与InnoDB

发表于 2020-05-28 | 分类于 数据库
  1. MyISAM是？
    MyISAM是MySQL的預設資料庫引擎（5.5版之前），由早期的ISAM所改良。 雖然效能極佳，但卻有一個缺點：不支援事务處理（transaction）。 不過，在這幾年的發展下，MySQL也導入了InnoDB（另一種資料庫引擎），以強化參考完整性與並行違規處理機制，後來就逐漸取代MyISAM。
  2. InnoDB是？
    InnoDB，是MySQL的数据库引擎之一，现为MySQL的默认存储引擎，为MySQL AB发布binary的标准之一。 InnoDB由Innobase Oy公司所开发，2006年五月时由甲骨文公司并购。
  3. 区别
    3.1 MyISAM表强调的是性能，其执行速度比InnoDB引擎更快，但不支持事务，而InnoDB表提供事务支持以及外部键等高级数据库功能。
    3.2 MySQL 5.6以前的版本，只有 MyISAM 存储引擎支持全文索引；MySQL 5.6 及以后的版本，MyISAM和InnoDB存储引擎均支持全文索引;只有字段的数据类型为char、varchar、text 及其系列才可以建全文索引。
    3.3 InnoDB表不保存表的具体行数，也就是说，执行select count(*) from table时，InnoDB表要扫描一遍整个表来计算有多少行，但是MyISAM表只要读出保存好的行数即可。值得注意的是，当上句sql包含where条件时，两种表的操作是一样的。
    3.4 对于AUTO_INCREMENT类型字段，InnoDB表必须包含只有该字段的索引，MyISAM表可以和其他字段一起建立联合索引。
    3.5 DELETE FROM table时，InnoDB引擎不会重新建立表，而是一行一行的删除。
    3.6 Innodb支持外键（转换一个有外键的InnoDB表为MYISAM时会失败）、行级锁（MyISAM支持最小粒度为表级锁）、事务处理。
    3.7 InnoDB是聚集索引，MyISAM是非聚集索引。
    3.8 每个MyISAM表在磁盘上存储成三个文件：1:定义表后缀为frm。2:数据文件后缀为.MYD。3:索引文件后缀为MYI。InnoDB表磁盘上存储为空间数据文件+日志文件。
    3.9 MyISAM更宜读，InnoDB更宜写。
    3.10 Innodb重全，MyISAM轻快。
  4. 命令
    4.1 现提供的存储引擎
    show engines;
    4.2 默认的存储引擎
    show variables like '%storage_engine%';
    4.3 表的存储引擎
    show table status from db_name where name='table_name'
    4.4 修改表引擎
    alter table user_info engine=INNODB;
    4.5 修改默认存储引擎：
    在mysql配置文件（linux下为/etc/my.cnf），在mysqld后面增加default-storage-engine=INNODB即可。
