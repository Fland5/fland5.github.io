Mysql优化55条

发表于 2020-06-01 | 分类于 数据库 | 评论数： | 阅读次数：
  1. 尽量不要在数据库中做运算。
  2. 控制单表数据量。
    建议单库不超300个表。
    建议单表数据/年不超500w行。
  3. 控制字段数。
    建议单表字段数不超20个。
  4. 平衡范式与冗余。
  5. 拒绝3B。
    5.1 Big SQL（大SQL）
    5.2 Big Transaction (大事务)
    5.3 Big Batch(大批量)
  6. 用好数值字段类型。
  7. 将字符转为数字。
  8. 优先使用ENUM或SET。
  9. 避免使用NULL字段。
  10. 少用并拆分TEXT/BLOB。
    若必须则拆分到单独的表
  11. 不在数据库里存图片。
  12. 合理添加索引
    不超字段数的20%。
  13. 字符串必须建立前缀索引。
  14. 不在索引列做运算。
  15. 自增或全局ID做INNODB主键。
  16. 尽量不用外键。
  17. 拒绝复杂SQL。
    一条SQL运行于一个CPU。
    简单SQL命中率更高。
    减少锁表时间，特别是MyIsam存储引擎。
  18. 保持事务短小。
  19. 尽可能避免使用SP/TRIG/FUNC。
  20. 尽量不用 SELECT *,需要什么取什么是个好习惯。
  21. 改写OR为IN()。
    OR效率：O(n)；
    IN 效率：O(Log n)；
    当n很大时，OR会慢很多；
    注意控制IN的个数，建议n小于200。
  22. 改写OR为UNION。
  23. 避免负向查询和%前缀模糊查询。
  24. COUNT(*)、COUNT(1)会包含null列。
  25. 减少COUNT(*)使用。
  26. LIMIT偏移量越大越慢。
  27. 用UNION ALL 而非 UNION。
  28. 分解联接保证高并发。
  29. GROUP BY 去除排序。
  30. 同数据类型的列值比较。
    数字对数字，字符对字符
  31. Load data 导数据
    Load data比insert快约20倍。
  32. 打散大批量更新。
  33. 隔离线上线下。
    实时数据用real库；
    模拟环境用sim库；
    测试用qa库；
    开发用dev库。
  34. 尽量少使用子查询。
  35. 永远不在程序端显式加锁。
  36. 统一字符集为UTF8。
    统一字符集：UTF8
    校对规则：utf8_general_ci
  37. 统一命名规范。
    MySQL库表大小写敏感；
    字段名的大小写不敏感。
  38. 注意避免用保留字命名。
  39. 查询缓存的使用。

curdate()、now()、rand()等函数因其返回值易变，因此不会开启查询缓存。

  1. explain一个select语句
    explain关键字可以让你知道mysql是如何处理你的sql语句的。
  2. 搜索字段建立索引。
  3. Join表时使用相同类型的列，并将其建立索引。String类型注意字符集要相同。
  4. 随机挑选一条数据可不要使用ORDER BY RAND()，因为ORDER BY RAND()耗时且会为每条数据执行一次，然后再排序，存在严重性能问题。
  5. 尽可能为数据库每张表设置一个数字类型自动增长的主键。
  6. 尽可能使用NOT NULL。
  7. 列越小越快，比如Date比DateTime要快，TINYINT比INT要快，适用且快捷为原则。
  8. 拆分大的INSERT、DELETE语句。
  9. 把IP地址存成 UNSIGNED INT。
  10. 所有字段固定长度的表会更快些。
  11. 由一变多，垂直分割。
  12. 隐式转换导致索引失效。
    假设id为字符类型
    select from user where id = 1;//索引失效
    select     from user where id = ‘1’;//索引有效
  13. 索引列运算(+ - / !)导致索引失效。
    select     from user where id-1 = 10;//索引失效
    select * from user where id = 10;//索引有效
  14. 如果变量可能为空，应使用 IS NULL 或 IS NOT NULL 进行比较，或使用ISNULL函数，而非比较运算符。
  15. 改sql中双引号为单引号。
